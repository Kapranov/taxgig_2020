    put_file(arg1)

    View Source 
    put_file(Core.Upload.t()) ::
  :ok | {:ok, file_spec()} | {:error, String.t()} | :wait_callback

    Put a file to the backend.

    Returns:

    • :ok which assumes {:ok, upload.path}

    • {:ok, spec} where spec is: {:file, filename :: String.t} to handle reads with get_file/1 (recommended)

      This allows to correctly proxy or redirect requests to the backend, while allowing to migrate backends without breaking any URL.

    • {url, url :: String.t} to bypass get_file/2 and use the url directly in the activity.

    • {:error, String.t} error information if the file failed to be saved to the backend.

    • :wait_callback will wait for an http post request at /api/community/upload_callback/:upload_path and call the uploader's http_callback/3 method.
